# Artifact Generator Module

Generates interactive HTML reports from knowledge pack analysis. Runs post-sync to provide operator visibility into URL references, content structure, and link health.

## Overview

The artifact generator module produces a self-contained, browser-viewable HTML dashboard that analyzes:

- **URL extraction**: All HTTP(s) links in source files
- **Reference ranking**: URLs sorted by frequency (impact scoring)
- **Severity classification**: High (10+ refs), Medium (5-9 refs), Low (1-4 refs)
- **Distribution analysis**: Chart of top 10 most-referenced URLs
- **KPI dashboard**: File count, unique URL count, avg refs per file, max references

## Usage

### NPM Scripts

```bash
# Generate artifact from NotebookLM knowledge pack
npm run artifact:generate

# Generate artifact from Obsidian staging directory
npm run artifact:generate:obsidian

# Generate both (sequential, fail-soft)
npm run artifact:generate:all
```

### Direct Invocation

```bash
bash modules/artifact-generator/generate.sh [config-file] [source]

# Example:
bash modules/artifact-generator/generate.sh configs/artifact-generator.yaml notebooklm
bash modules/artifact-generator/generate.sh configs/artifact-generator.yaml obsidian
```

### Programmatic (Node.js)

```bash
node modules/artifact-generator/generate-report.mjs --source notebooklm --config-file configs/artifact-generator.yaml
```

## Input Sources

### NotebookLM
- **Source**: `.nlm_pack/*.txt` (knowledge pack files generated by Stage 1)
- **Format**: Structured text with `--- START FILE:` / `--- END FILE:` delimiters
- **Availability**: After `npm run kb:sync:notebooklm` completes

### Obsidian
- **Source**: `$OBSIDIAN_VAULT_ROOT/_kb-sync-staging/<repo>/<timestamp>/` (staged files)
- **Format**: Raw markdown + other source files
- **Availability**: After `npm run kb:sync:obsidian` completes

## Configuration

Edit `configs/artifact-generator.yaml`:

```yaml
# Output location (relative to repo root)
output_dir: _integration

# Report filename
report_filename: kb-sync-interactive-report.html

# Enable URL extraction and analysis
url_analysis_enabled: true

# Maximum URLs to display in report (rest are truncated)
max_urls_displayed: 100

# Severity thresholds (reference count)
severity:
  high: 10    # >= 10 references → red
  medium: 5   # >= 5 references → orange
  low: 1      # < 5 references → green

# Future: Link validation (experimental)
link_validation:
  enabled: false
  timeout_seconds: 5
  concurrent_checks: 10
```

## Output

### File Location

Namespaced per source so NotebookLM and Obsidian reports coexist:
`_integration/kb-sync-interactive-report-notebooklm.html`

### Features
- **Self-contained**: No external dependencies (Chart.js loaded from CDN)
- **Light/Dark theme**: Respects OS preference via `prefers-color-scheme`
- **Interactive charts**: Hover for data, responsive layout
- **Sortable table**: Top 100 URLs ranked by reference count
- **Severity badges**: Color-coded link severity
- **KPI cards**: Quick metrics overview

## Integration with Multi-Target Sync

When `npm run kb:sync:all` executes:

1. **NotebookLM target** runs → generates `.nlm_pack/repo_knowledge_pack.txt`
2. **Obsidian target** runs → stages files to vault `_kb-sync-staging/`
3. **Post-sync** (if all succeeded):
   - Artifact generated from NotebookLM pack
   - Artifact generated from Obsidian staging (separate report)
   - Both artifacts output to `_integration/`

If either sync fails, artifact generation is **skipped for that target** (fail-soft). The operator can still generate artifacts manually using npm scripts.

## Error Handling

### Source Not Found
```
[ARTIFACT-GENERATOR] [ERROR] Pack directory not found: ./.nlm_pack
Run 'npm run kb:sync:notebooklm' first to generate knowledge pack.
```

### Node.js Not Available
```
[ARTIFACT-GENERATOR] [ERROR] Node.js not found. Cannot generate artifact.
```

### Config File Missing
Falls back to hardcoded defaults (output_dir: `_integration`, report_filename: `kb-sync-interactive-report.html`)

## Performance

- **Typical runtime**: 1-5 seconds (depends on pack size)
- **Memory**: O(n) where n = number of URLs extracted
- **I/O**: Single read pass over source files, single write of HTML

## Future Enhancements

- **Link validation**: HEAD requests to detect broken links (enabled via config)
- **Trend tracking**: Historical CSV log of sync metrics
- **Integration**: Slack/email notifications with artifact link
- **Formats**: PDF export, Markdown version for git storage

## Troubleshooting

### Artifact not generated after sync
1. Check: `npm run kb:sync:all` succeeded (check exit code)
2. Verify: `ls _integration/kb-sync-interactive-report.html`
3. Run manually: `npm run artifact:generate`
4. Check logs for errors (grep `[ARTIFACT-GENERATOR] [ERROR]`)

### Reports are empty or have no URLs
1. Check source files contain HTTP links
2. Verify URL regex isn't too restrictive (search pattern: `https?://[^\s)"\]]+`)
3. Manually inspect pack file: `head -100 .nlm_pack/repo_knowledge_pack.txt`

### Obsidian artifact references wrong vault
1. Verify: `echo $OBSIDIAN_VAULT_ROOT`
2. Check `.env` has correct path: `grep OBSIDIAN_VAULT_ROOT .env`
3. Verify staging directory exists: `ls $OBSIDIAN_VAULT_ROOT/_kb-sync-staging/`
